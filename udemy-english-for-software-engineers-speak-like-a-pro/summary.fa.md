## قبل از شروع
- من نکات دوره‌های کاربردی را خلاصه می‌کنم تا سریع‌تر یاد بگیرید و مرور کنید.

## پرسیدن درباره جزئیات پروژه

* **خلاصه**: چطور درباره اولویت‌ها، feedback loop و timeline هنگام پیوستن به یک پروژه سؤال بپرسیم.
* **مثال**: مثال: «Timeline نمونه اولیه چقدر است؟» → «هدف ما شش هفته با weekly check‑in برای blockers است.»
* **لینک برای جزئیات بیشتر**: [Ask AI: پرسیدن درباره جزئیات پروژه](https://alisol.ir/?ai=%D9%BE%D8%B1%D8%B3%DB%8C%D8%AF%D9%86%20%D8%AF%D8%B1%D8%A8%D8%A7%D8%B1%D9%87%20%D8%AC%D8%B2%D8%A6%DB%8C%D8%A7%D8%AA%20%D9%BE%D8%B1%D9%88%DA%98%D9%87%7CTerry%20Mc%20Gonigle%7CEnglish%20for%20Software%20Engineers%3A%20Boost%20Your%20Global%20Tech%20Career)

## مقدمه الگوهای سؤال‌سازی در انگلیسی

* **خلاصه**: هفت الگوی پرکاربرد در مکالمات فنی (What / How / How often / How long / Can–Could / What + technology / Is there–Are there).
* **مثال**: مثال: «How often security audit انجام می‌دهیم؟» → «Quarterly و بعد از major releaseها.»
* **لینک برای جزئیات بیشتر**: [Ask AI: مقدمه الگوهای سؤال‌سازی در انگلیسی](https://alisol.ir/?ai=%D9%85%D9%82%D8%AF%D9%85%D9%87%20%D8%A7%D9%84%DA%AF%D9%88%D9%87%D8%A7%DB%8C%20%D8%B3%D8%A4%D8%A7%D9%84%E2%80%8C%D8%B3%D8%A7%D8%B2%DB%8C%20%D8%AF%D8%B1%20%D8%A7%D9%86%DA%AF%D9%84%DB%8C%D8%B3%DB%8C%7CTerry%20Mc%20Gonigle%7CEnglish%20for%20Software%20Engineers%3A%20Boost%20Your%20Global%20Tech%20Career)

## اجایل و اسکرام

* **خلاصه**: واژگان مربوط به مراسم‌ها و فرکانس: daily stand‑up، sprint planning/review/retro، backlog refinement.
* **مثال**: مثال: «Retrospectiveها هر چند وقت برگزار می‌شوند؟» → «انتهای هر sprint؛ هر ۲–۴ هفته.»
* **لینک برای جزئیات بیشتر**: [Ask AI: Agile و Scrum](https://alisol.ir/?ai=Agile%20%D9%88%20Scrum%7CTerry%20Mc%20Gonigle%7CEnglish%20for%20Software%20Engineers%3A%20Boost%20Your%20Global%20Tech%20Career)

## ویژن  و اسکوپ

* **خلاصه**: چطور درباره اهداف big‑picture و محدوده فنی (از محدود تا گسترده) سؤال کنیم و focus areas را بیان کنیم.
* **مثال**: مثال: «می‌توانید یک overview از vision و scope پروژه بدهید؟» → «یک پرتال املاک user‑friendly با interactive map و AI recommendations.»
* **لینک برای جزئیات بیشتر**: [Ask AI: Vision و Scope](https://alisol.ir/?ai=Vision%20%D9%88%20Scope%7CTerry%20Mc%20Gonigle%7CEnglish%20for%20Software%20Engineers%3A%20Boost%20Your%20Global%20Tech%20Career)

## شفاف‌سازی نیازمندی‌ها

* **خلاصه**: راه‌های مودبانه برای اعلام ابهام و درخواست جزئیات؛ ساختار سؤال‌های either/or؛ افعالی مثل clarify، normalize/standardize/cleanse data.
* **مثال**: مثال: «برای inventory tracking، data را از supplier API می‌گیریم یا یک central DB؟»
* **لینک برای جزئیات بیشتر**: [Ask AI: شفاف‌سازی نیازمندی‌ها](https://alisol.ir/?ai=%D8%B4%D9%81%D8%A7%D9%81%E2%80%8C%D8%B3%D8%A7%D8%B2%DB%8C%20%D9%86%DB%8C%D8%A7%D8%B2%D9%85%D9%86%D8%AF%DB%8C%E2%80%8C%D9%87%D8%A7%7CTerry%20Mc%20Gonigle%7CEnglish%20for%20Software%20Engineers%3A%20Boost%20Your%20Global%20Tech%20Career)

* **مثال**: مثال: «در React از کتابخانه خاصی استفاده می‌کنیم؟» → «Redux برای state و Router برای navigation.»

* **مثال**: مثال: «Testها را در CI/CD ادغام کرده‌ایم؟» → «بله—Selenium روی هر PR در Jenkins اجرا می‌شود.»

## نیازمندی‌های محصول

* **مثال**: مثال: «پلتفرم خاصی مدنظر داریم؟» → «WordPress/Shopify برای انعطاف e‑commerce.»
